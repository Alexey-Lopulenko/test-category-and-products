Shop-TEST — каталог товарів (чистий PHP + MySQL + Docker)

Тестове завдання: категорії + товари, AJAX-фільтрація та сортування без перезавантаження, Bootstrap модалка “Купити”.

PHP 8.3 MySQL 8.0 Docker Compose Bootstrap 5 Vanilla JS (fetch) utf8mb4 OOP

Функціонал

  • Список категорій зліва (біля кожної — кількість товарів).
  • Список товарів справа.
  • Клік по категорії — AJAX підвантаження товарів цієї категорії.
  • Сортування: спочатку дешевші, по алфавіту, спочатку нові — без перезавантаження.
  • “Купити” → Bootstrap модалка з вибраним товаром.
  • Стан в URL: ?category_id=... та ?sort=... — відновлюється після reload.
  • PHP → JS дані передаються як JSON (window.__INITIAL_STATE__).

Технології

  • PHP 8.3 (Apache)
  • MySQL 8.0
  • Bootstrap 5
  • Vanilla JS (fetch API)
  • Docker + Docker Compose
  • UTF-8 / utf8mb4 (щоб не було “кракозябр”)

🚀 Запуск

З кореня проєкту:

docker compose up -d --build

Відкрити:

http://localhost:8080

⚙️ Конфігурація (.env.php)

Файл .env.php в корені:

<?php

return [
  'db' => [
    'dsn'  => 'mysql:host=db;port=3306;dbname=catalog_test;charset=utf8mb4',
    'user' => 'app',
    'pass' => 'app',
    'opt'  => [],
  ],

🔌 API

Категорії

GET /api/categories.php

Повертає список категорій + кількість товарів в кожній.

Товари

GET /api/products.php?category_id=1&sort=cheap

Параметри:
category_id (0 або відсутній = всі)
sort = cheap | alpha | newest

Один товар (для модалки)

GET /api/product.php?id=5

🧠 Frontend логіка

  • Стан сторінки (категорія + сортування) відновлюється з URL.
  • PHP передає стартовий стан в JS як JSON:
<script>
window.__INITIAL_STATE__ = {"category_id":1,"sort":"newest"};
</script>
  • При зміні категорії/сортування:
    • робиться fetch до API
    • оновлюється DOM
    • оновлюється URL через history.pushState()
